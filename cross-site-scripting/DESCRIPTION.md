If a web server takes user-controlled input and inserts it into the response without properly handling it, the browser may treat that input as elements or even worse, scripts instead of text data.

The class of vulnerabilities in which injections occur into client-side web data (such as HTML) is called _Cross-Site Scripting_, or XSS.

There are three types of XSS:

- DOM-based (Type 0)
- Reflected (Type 1)
- Stored (Type 2)
